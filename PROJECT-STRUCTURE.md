## File Descriptions

### Configuration Files

#### `neutralino.config.json`
**Purpose:** Main configuration for Neutralino.js
**Contains:**
- Application ID and version
- Window settings (size, title, icon)
- Native API permissions
- Server configuration
- Build settings

**Key sections:**
```json
{
  "applicationId": "com.texteditor.minimal",
  "version": "1.0.0",
  "modes": {
    "window": {
      "title": "Minimal Text Editor",
      "width": 1000,
      "height": 700
    }
  }
}
```

#### `package.json`
**Purpose:** NPM package metadata
**Contains:**
- Project name and version
- NPM scripts for common tasks
- Development dependencies

**Usage:**
```bash
npm run dev    # Run in development
npm run build  # Build executable
```

#### `.gitignore`
**Purpose:** Tells Git which files to ignore
**Excludes:**
- Build artifacts (.tmp/, dist/)
- Auto-generated files
- System files
- IDE files

### Scripts (Windows Batch Files)

#### `setup.bat`
**Purpose:** Automated project setup
**Does:**
1. Checks for Node.js
2. Installs Neutralino CLI if needed
3. Downloads Neutralino binaries
4. Offers to run or build

**Usage:** Double-click or run `setup.bat`

#### `run.bat`
**Purpose:** Quick run in development mode
**Does:** Starts the app with `neu run`

**Usage:** Double-click or run `run.bat`

#### `build.bat`
**Purpose:** Quick build to executable
**Does:**
1. Builds the app
2. Shows build location
3. Offers to open build folder

**Usage:** Double-click or run `build.bat`

### Application Source Files

All source files are in the `resources/` directory.

#### `resources/index.html`
**Purpose:** Main HTML structure
**Contains:**
- Toolbar with icons
- Settings panel
- Editor textarea
- SVG icons inline
- Script imports

**Key features:**
- Semantic HTML5
- Inline SVG icons (no external files)
- Minimal markup for fast loading

#### `resources/style.css`
**Purpose:** All application styles
**Contains:**
- CSS variables for themes
- Layout styles
- Component styles
- Responsive design
- Dark theme support

**Structure:**
```css
/* Theme Variables */
body[data-theme="dark"] { ... }

/* Layout */
.toolbar { ... }
.editor-container { ... }

/* Components */
.icon-btn { ... }
.settings-panel { ... }
```

**Size:** ~8 KB (highly optimized)

#### `resources/script.js`
**Purpose:** Application logic
**Contains:**
- Neutralino.js integration
- File operations (open/save)
- Settings management
- Auto-save functionality
- Keyboard shortcuts
- Storage management

**Key features:**
- Native file dialogs via Neutralino API
- Persistent storage via Neutralino storage API
- Async/await for clean code
- Error handling

**Size:** ~10 KB

## Important Notes

### Don't Edit These Files/Folders:
- `.tmp/` - Build artifacts
- `dist/` - Final build output
- `resources/js/neutralino.js` - Auto-generated

### Safe to Edit:
- `resources/index.html` - UI structure
- `resources/style.css` - Styles
- `resources/script.js` - Logic
- `neutralino.config.json` - Settings
- `resources/icons/appIcon.png` - Icon

### Must Stay Together (for distribution):
```
minimal-text-editor.exe
resources.neu
WebView2Loader.dll
```
These three files MUST be in the same folder.

## Workflow

### Development Workflow:
```
1. Edit files in resources/
2. Run: neu run
3. Test changes
4. Repeat
```

### Build Workflow:
```
1. Make final changes
2. Run: neu build
3. Test dist/minimal-text-editor/minimal-text-editor.exe
4. Distribute dist/minimal-text-editor/ folder
```

### Distribution:
```
1. Build the app
2. ZIP the dist/minimal-text-editor/ folder
3. Share the ZIP
4. Users extract and run .exe
```

## Adding New Features

### To add a new feature:

1. **Edit HTML** (`resources/index.html`):
   - Add new UI elements
   - Add new buttons/controls

2. **Edit CSS** (`resources/style.css`):
   - Style new elements
   - Add new themes/colors

3. **Edit JavaScript** (`resources/script.js`):
   - Add new functions
   - Wire up event listeners
   - Use Neutralino APIs if needed

4. **Test:**
   ```bash
   neu run
   ```

5. **Build:**
   ```bash
   neu build
   ```
